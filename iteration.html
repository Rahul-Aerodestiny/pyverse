Iterations in Python

Master looping techniques: for, while, enumerate, zip, nesting, controls and comprehensions.

Try it Yourself →

Why Iteration?

Iteration lets your program repeat tasks. Learn idiomatic patterns — they make code shorter, faster and easier to read.

Basic for Loop

fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

Iterates directly over elements — Pythonic and simple.

The while Loop

n = 1
while n <= 5:
    print("n =", n)
    n += 1

Use when the number of iterations depends on a condition.

Using range()

for i in range(3):        # 0,1,2
    print("Iteration", i)

for i in range(1, 6):     # 1..5
    print(i)

Great for index-based loops and numeric sequences.

enumerate() — index + value

names = ["Amar", "Akbar", "Anthony"]
for idx, name in enumerate(names, start=1):
    print(idx, name)

Use start= to begin indexing from any number.

zip() — iterate in parallel

subjects = ["Math", "Science", "English"]
marks    = [90, 85, 88]

for subject, mark in zip(subjects, marks):
    print(subject, ":", mark)

Stops at the shortest sequence; use itertools.zip_longest() for full coverage.

Nested Loops

matrix = [
  [1, 2, 3],
  [4, 5, 6]
]

for row in matrix:
    for val in row:
        print(val)

Useful for 2D data; be mindful of performance for deep nesting.

Loop Control: break, continue, pass

for i in range(6):
    if i == 2:
        continue    # skip 2
    if i == 5:
        break       # stop entirely
    print(i)

# pass is a placeholder
for _ in range(3):
    pass

Use control keywords to shape loop behaviour precisely.

List / Set / Dict Comprehensions

# List comprehension
squares = [x*x for x in range(6)]

# Conditional comprehension
evens = [x for x in range(10) if x % 2 == 0]

# Dict comprehension
squares_map = {x: x*x for x in range(6)}

Comprehensions are compact and fast — prefer them for simple transformations.

Iterator Protocol & Generators (brief)

# A simple generator
def gen_numbers(n):
    for i in range(n):
        yield i

for num in gen_numbers(3):
    print(num)

Generators yield values lazily and save memory for large sequences.

Tips & Common Pitfalls

  • Avoid modifying a list you’re iterating over — iterate over a copy instead.
  • Zip stops at the shortest input; use zip_longest if needed.
  • Use comprehensions for readability, but don’t cram complex logic into a single line.
  • Prefer enumerate over manual index tracking (more Pythonic).
