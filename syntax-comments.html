Python Syntax: Comments

Learn how to use comments in Python code to make your programs more readable and maintainable.

Back to Basics →

What are Comments?

Comments are lines in Python code that are ignored by the interpreter. They help explain your code, make it easier to read, or temporarily disable code during testing.

Single-line Comments

Syntax: Use # to start a comment. Everything after # on that line is ignored. 
# This is a single-line comment
print("Hello, World!")  # This comment is after code

Multi-line Comments

Python does not have a dedicated multi-line comment syntax. You can use several # lines, or use a multi-line string (triple quotes) as a comment.

Example: 
# This is a comment
# that spans multiple lines

"""
This is a multi-line string,
often used as a multi-line comment.
"""
Note: Triple-quoted strings are not technically comments, but if not assigned to a variable, they are ignored by the interpreter.

Best Practices for Comments

  • Use comments to explain why something is done, not what is done.
  • Keep comments up-to-date with code changes.
  • Write clear code so you need fewer comments.
  • Don't overuse comments—focus on readability.
